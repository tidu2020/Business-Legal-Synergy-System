## 4.3 集成测试（端到端闭环）

`tests/integration_test.py` 模拟真实用户操作，覆盖完整业务-法务闭环：

### 测试流程

```
1. 业务账号登录
2. 发起 AI 咨询 → 验证返回四段式意见 + 免责声明
3. 上传材料（.txt/.docx）→ 验证解析成功
4. 提交法务工单 → 验证工单创建
5. 法务账号登录
6. 查看工单列表 → 验证工单可见
7. 开始审核 → 验证状态流转
8. 调用知识库检索 → 验证法务可调用
9. 提交拟入库条目 → 校验（含重复检测）
10. 确认入库 → 验证知识库新增
11. 归档 → 验证归档包生成
```

| 3 | POST /api/business/upload | ✅ 200 | txt/docx 均解析成功 |
| 4 | POST /api/business/submit_order | ✅ 200 | 工单 ID 生成 |
| 5 | POST /api/auth/login（法务） | ✅ 200 | - |
| 6 | GET /api/legal/orders | ✅ 200 | 工单列表返回 |

| 8 | POST /api/legal/search_kb | ✅ 200 | 法务检索命中 |

| 10 | POST /api/legal/confirm | ✅ 200 | 知识库 +1 |
| 11 | POST /api/archive/orders/{id}/archive | ✅ 200 | 归档包生成 |

**集成测试结论**：11 步全部通过，闭环流程完整。

## 4.6 问题与修复记录

| # | 问题描述 | 原因 | 修复方案 | 状态 |
|---|---------|------|---------|------|
| 1 | LLM API key 格式错误（HTTP 401） | key 格式不符合 Ark 要求 | 用户提供正确 key（UUID 格式）；代码层增加回退机制 | ✅ 已修复 |
| 2 | 端口 5000 被占用 | 多进程监听同一端口 | taskkill 清理后重启 | ✅ 已修复 |
| 3 | KnowledgeBase 无 status_breakdown 方法 | /api/kb/stats 缺方法 | 在 models.py 补充 status_breakdown() | ✅ 已修复 |
| 4 | 集成测试重复入库冲突（HTTP 409） | 重复校验触发二次确认 | 测试脚本支持 force=True 重试 | ✅ 已修复 |
| 5 | __pycache__ 导致旧代码残留 | Python 缓存 | 重启前清理 __pycache__ | ✅ 已修复 |
| 6 | file_review 内部 import 失败 | 移植后包路径变化 | `from src.` 改为 `from .src.` 相对导入 | ✅ 已修复 |
| 7 | file_review 依赖外部 `openai` 库 | 原项目用 openai SDK | 重写 `llm_client.py` 用 `urllib` 标准库实现 | ✅ 已修复 |
| 8 | 前端调用已删除的 `/api/ai/contract_upload` | 旧端点移除 | 更新为 `/api/file_review/upload`，UI 同步刷新 | ✅ 已修复 |
| 9 | **business review failed: 'str' object has no attribute 'get'** | LLM 返回的 `clause_reviews` 数组中含字符串元素而非对象 | 在 `_parse_review_result` 中加 `isinstance(c, dict)` 防御，跳过非对象并记录警告 | ✅ 已修复 |
| 10 | LLM API key 未配置 | 环境变量未注入 | 设置 `LLM_API_KEY`/`LLM_BASE_URL`/`LLM_MODEL` 环境变量 | ✅ 已修复 |
