# 07 系统更新计划

---

## 概述

本文件汇总业法协同系统的后续迭代计划，涵盖从 MVP 到生产部署的全部待开发功能。计划按优先级分为三个级别：

| 优先级 | 说明 |
|--------|------|
| **高** | 生产环境必备，影响核心功能稳定性和安全性 |
| **中** | 提升用户体验和系统能力，增强核心价值 |
| **低** | 锦上添花功能，可根据实际需求延后 |

---

## 高优先级

### 1. 生产环境切换私有部署 LLM

**当前状态**：MVP 使用测试大模型（ark-code-latest）

**目标**：通过环境变量注入公司私有部署的大模型配置

**实现方案**：
- 通过 `config.py` 的环境变量优先机制注入：
  - `LLM_API_KEY`：私有模型密钥
  - `LLM_BASE_URL`：私有模型地址
  - `LLM_MODEL`：私有模型名称
- 确保涉密数据不出公司网络

**依赖**：公司私有大模型部署完成

---

### 2. 自动判断入库申请是否确认

**当前状态**：入库申请需要管理员手动批准/驳回

**目标**：二次 LLM 评审 + 规则匹配，自动批准/驳回入库申请

**实现方案**：
- 调用 LLM 对入库申请进行二次评审
- 结合规则匹配（如相似度阈值、字段完整性等）
- 自动批准符合条件的申请，标记需要人工审核的申请

**依赖**：知识库入库申请流程稳定运行

---

---

## 中优先级

### 3. 风险监测引擎

**当前状态**：无主动风险识别能力

**目标**：定时扫描知识库，按月/季聚合识别高发问题

**实现方案**：
- 新增 `RiskMonitor` 类
- 调用统计 + 聚类算法分析知识库条目
- 识别近期高发法律问题和风险趋势
- 生成风险监测报告

**依赖**：知识库数据积累到一定规模

---

### 4. 年度风险报告

**当前状态**：无年度风险报告功能

**目标**：基于风险监测数据，生成年度风险报告

**实现方案**：
- 基于风险监测引擎的数据
- 调用 `ReportExporter` 生成结构化年度报告
- 包含风险趋势、高发问题、改进建议等内容

**依赖**：风险监测引擎实现

---

### 5. 审核留痕与审计日志（AuditLog）

**当前状态**：WorkOrder 已记录关键节点的操作人和时间戳，但未记录完整状态变更链

**目标**：实现完整审计日志，满足企业法务合规要求

**实现方案**：
- 新增 `AuditLog` 类，记录工单每一次状态变更：
  ```json
  {
    "work_order_id": "WO-2026-0001",
    "event": "status_change",
    "from_state": "ai_reviewed",
    "to_state": "submitted_to_legal",
    "operator": "business01",
    "timestamp": "2026-06-25T10:30:00",
    "detail": "业务提交法务工单"
  }
  ```
- WorkOrder 的所有状态变更方法自动写入 AuditLog
- 支持按工单 ID 查询完整操作轨迹
- 支持按操作人/时间范围查询（满足审计检查场景）

**触发场景**：
- 内部合规检查：调取某工单完整操作链
- 责任追溯：某入库条目争议时，回溯来源工单的审核全过程
- 监管报送：年度法治工作总结时，统计审核流转数据

## 低优先级

### 13. 结构化数据抽取

**当前状态**：审核结论以 Q&A 形态入库

**目标**：增加关系库字段抽取（合同主体、金额、期限、风险等级）

**实现方案**：
- 新增 `StructuredExtractor` 类
- 调用 LLM 抽取关键字段入 SQLite
- 支持结构化查询和统计分析

---

### 14. 知识出版

**当前状态**：支持 Markdown 导出

**目标**：按主题/案例聚合，自动生成"典型案例集"书稿

**实现方案**：
- 新增 `BookCompiler` 类
- 按 `category` + `case_id` 聚合相似案例
- 自动生成书稿格式（支持 PDF/EPUB 导出）

---

### 15. 电子档案归档（ArchivePackager）

**当前状态**：工单数据以 JSON 散存于 work_orders 目录，无统一归档

**目标**：法务确认入库后，系统自动整理电子档案归档

**归档触发流程**：
1. 法务确认入库
2. 系统自动打包：工单数据 + 对话记录 + 附件材料 + 审核结论 + 入库条目
3. 生成标准化电子档案包（ZIP 格式）
4. 存储至归档目录，支持按月份/案例检索

**档案包结构**：
```
archive/
├─ WO-202606-001/
│   ├─ metadata.json          # 工单元数据
│   ├─ dialogue.md            # 对话记录
│   ├─ materials/             # 附件材料
│   │   ├─ contract.docx
│   │   └─ memo.txt
│   ├─ conclusion.md          # 审核结论
│   └─ faq.json               # 入库条目
└─ index.json                 # 归档索引
```
