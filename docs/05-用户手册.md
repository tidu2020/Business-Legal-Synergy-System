## 5.1 安装与部署

### 5.1.1 环境要求

| 项 | 要求 |
|---|---|
| 操作系统 | Windows 10/11、macOS、Linux |
| Python | 3.9 及以上 |
| 浏览器 | Chrome / Edge / Firefox（现代浏览器） |
| 依赖 | Flask（`pip install flask`） |

### 5.1.2 安装步骤

```bash
# 1. 进入项目目录
cd d:\Users\Z\Desktop\业法协同系统

# 2. 安装依赖
pip install flask

# 3.（可选）创建虚拟环境
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS/Linux
```

### 5.1.3 启动系统

```bash
python -m backend.app
```

启动成功后控制台显示：

```
[LLM] 已启用：ark-code-latest @ https://ark.cn-beijing.volces.com/api/coding/v3
============================================================
业法协同系统 — 启动中
访问地址：http://127.0.0.1:5000
============================================================
 * Running on http://127.0.0.1:5000
```

浏览器访问 `http://127.0.0.1:5000/` 进入登录页。

### 5.1.4 大模型配置（可选）

系统默认使用测试用大模型配置。生产环境请通过环境变量覆盖：

```bash
# Windows PowerShell
$env:LLM_API_KEY="你的私有部署key"
$env:LLM_BASE_URL="https://你的私有部署地址/v1"
$env:LLM_MODEL="你的模型名"

# Linux/macOS
export LLM_API_KEY="你的私有部署key"
export LLM_BASE_URL="https://你的私有部署地址/v1"
export LLM_MODEL="你的模型名"
```

**不配置大模型也能用**：系统会自动降级为纯 RAG 模式（基于规则拼装四段式回复），功能完整，仅回复风格不如 LLM 自然。

## 5.3 业务用户操作指南

### 5.3.1 发起法务咨询

1. 用 `business01` 登录，进入业务端
2. 在对话框输入问题，例如：
   - "合同相对方要求第三方付款可以吗"
   - "证据逾期提交有什么后果"
   - "公众号使用网络图片侵权吗"
3. 点击发送，等待 AI 回复（约 3-8 秒）
4. 系统返回**四段式法务意见**：
   - 法律解答
   - 合规风险
   - 实操建议
   - 相关法条
5. 可在同一会话内继续追问，支持多轮对话

> ⚠️ 每条 AI 回复均附免责声明，仅供参考，不可作为正式决策依据。

### 5.3.2 上传材料

1. 点击"上传材料"按钮
2. 选择文件（支持 `.txt`、`.docx`）
3. 系统自动解析并存入当前会话
4. 上传的材料会随工单一并提交给法务

### 5.3.3 提交法务工单

1. 咨询完毕后，点击"提交法务处理"
2. 系统打包：咨询对话 + 上传材料 + AI 结论
3. 生成工单号，进入"待法务审核"状态
4. 可在"我的工单"查看进度

### 5.3.4 工单状态流转

```
待审核 → 审核中 → 已确认入库 → 已归档
   ↓
 已撤回（业务可在法务开始审核前撤回）
```

**撤回工单**：在"我的工单"中，对状态为"待审核"的工单点击"撤回"按钮即可撤回。法务一旦开始审核则无法撤回。

### 5.3.5 合同审核（新增）

业务端顶部新增"合同审核"标签页，提供独立的合同审核能力：

1. **上传合同**：点击"上传合同"按钮，选择 `.txt` / `.md` / `.docx` 合同文件
2. **启动审核**：填写"特别关注"（可选）和"审核立场"（甲方/乙方/中立/自动识别），点击"启动审核"
3. **等待流水线**：系统执行六阶段审核（约 30-60 秒），控制台实时显示进度
   - 阶段 1：合同信息提取
   - 阶段 2：知识检索（案由匹配 + 法律检索）
   - 阶段 3：业务审查（商务条款、付款、违约责任）
   - 阶段 4：法务审查（法律效力、合规风险）
   - 阶段 5：法条核验（高风险法条 LLM 复核）
   - 阶段 6：报告生成（Markdown）
4. **查看报告**：审核完成后，报告预览自动展示在页面下方
5. **下载报告**：点击"下载报告"按钮，保存为 `.md` 文件
6. **多轮问答**：在合同审核页底部对话框，针对合同细节继续提问（如"违约金条款合理吗"），系统基于合同全文 + 已有报告作答
7. **会话持久化**：刷新页面或重启服务后会话不丢，可在"会话列表"中查看历史

> ⚠️ 合同审核模块独立运行，不依赖知识库；需要配置 `LLM_API_KEY` 才能执行审核，未配置时上传/会话仍可用，但 `/run` 会返回 503。

## 5.4 法务用户操作指南

### 5.4.1 接收并审核工单

1. 用 `legal01` 登录，进入法务端
2. 左侧"待办列表"显示待审核工单
3. 点击工单，查看：
   - 业务咨询对话
   - 上传材料内容
   - AI 生成的结论
4. 点击"开始审核"，工单状态变为"审核中"

### 5.4.2 调用知识库检索

1. 在审核界面使用"知识库检索"功能
2. 输入关键词，查看相关历史案例
3. 作为审核参考（法务不可直接修改知识库，只能检索调用）

### 5.4.3 归档前校验（核心）

法务对拟入库条目修改后，点击"校验"，系统自动执行 6 项检查：

| 校验项 | 类型 | 失败处理 |
|--------|------|---------|
| 问题完整性 | 硬 | 阻止入库 |
| 四段式完整性 | 硬 | 阻止入库 |
| 法条格式 | 软 | 警告，可强制 |
| 标签完整性 | 软 | 警告，可强制 |
| 重复校验 | 软 | 警告，显示相似条目 |
| 问题格式 | 软 | 警告，可强制 |

- **硬错误**：必须修改后才能入库
- **软警告**：可勾选"强制入库"二次确认

### 5.4.4 确认入库

1. 校验通过（或强制确认）后，点击"确认入库"
2. 条目写入知识库，`source_work_order_id` 记录来源工单
3. 建立完整追溯链：知识 → 工单 → 原始材料

### 5.4.5 修改已归档内容（新增）

法务可对自己归档的工单进行重新编辑：

1. 在法务端"归档列表"中找到目标工单
2. 点击"重新编辑归档"按钮，工单状态从"已归档"回到"审核中"
3. 修改法务结论或入库条目后，重新走校验 → 确认入库 → 归档流程

### 5.4.6 提交知识库修改申请（新增）

法务发现知识库条目有误时，可提交修改申请：

1. 在法务端"知识库检索"中找到目标条目
2. 点击"提交修改申请"，填写修改内容与理由
3. 申请进入"待管理员审批"状态
4. 管理员审批通过后修改生效；驳回则保持原状

### 5.4.7 调用 AI 审核工具（新增）

法务在审核工单时可直接调用 AI 辅助：

1. 在工单详情页点击"AI 辅助审核"
2. 系统将工单材料送入 AI 引擎，返回四段式意见
3. 法务参考 AI 意见后撰写最终结论

## 5.5 管理员操作指南

### 5.5.1 知识库管理

1. 用 `admin01` 登录，进入管理员端
2. "知识库管理"页面可：
   - 查看全部条目（支持按月份/分类/状态筛选）
   - 编辑条目
   - 删除条目
   - 查看统计：总数、月份分布、分类分布、状态分布

### 5.5.2 归档导出

1. 进入"归档管理"
2. 选择已确认的工单
3. 点击"归档"，生成 Markdown 归档包：
   - 工单号、提交人、提交时间
   - 业务咨询完整对话
   - 上传材料
   - AI 结论
   - 法务结论
   - 入库条目

### 5.5.3 账户管理

管理员可查看所有账户，管理角色权限。

### 5.5.4 退回归档（新增）

管理员发现归档内容有问题时，可退回归档：

1. 在管理员端"归档管理"中找到目标工单
2. 点击"退回归档"，填写退回原因
3. 工单状态从"已归档"回到"审核中"，法务端收到重新编辑通知
4. 法务修改后重新归档

### 5.5.5 相似知识合并（新增）

系统自动检测相似度较高的知识库条目并提示合并：

1. 在管理员端"知识库管理"中，系统会高亮显示相似条目对
2. 点击"合并"按钮，预览合并效果
3. 选择主条目，将次条目内容合并入主条目
4. **合并需法务确认**：系统会通知法务复核合并结果，法务确认后才会真正删除次条目

### 5.5.6 审批知识库修改申请（新增）

1. 在管理员端"修改申请审批"中查看法务提交的修改申请
2. 查看原内容与修改内容对比
3. 点击"通过"或"驳回"
4. 通过后修改立即生效；驳回需填写驳回理由

## 5.6 常见问题（FAQ）

### Q1：启动后端报"端口被占用"？

```bash
# 查找占用 5000 端口的进程
netstat -ano | findstr :5000
# 终止进程（替换 PID）
taskkill /PID <PID> /F
# 重启
python -m backend.app
```

### Q2：AI 回复很慢？

- 纯 RAG 模式（LLM 未配置）：< 1 秒
- RAG+LLM 模式：3-8 秒（大模型生成耗时）
- 如需更快，可关闭 LLM（删除 config.py 中的 key），使用纯 RAG

### Q3：LLM 调用失败怎么办？

系统会自动回退到纯 RAG 模式，不影响使用。检查项：
- API key 是否正确（Ark key 为 UUID 格式）
- 网络是否能访问 `https://ark.cn-beijing.volces.com`
- `LLM_BASE_URL` 是否正确

### Q4：修改代码后不生效？

清理 Python 缓存后重启：

```bash
# Windows PowerShell
Get-ChildItem -Path . -Recurse -Directory -Filter __pycache__ | Remove-Item -Recurse -Force
python -m backend.app
```

### Q5：如何重置知识库？

删除 `data/knowledge_base.json` 后重启，系统会从 `FAQ整理-最终版.md` 重新加载。

### Q6：支持哪些文件上传？

- `.txt`：直接读取
- `.docx`：通过 `python-docx` 读取（需 `pip install python-docx`）
- 其他格式暂不支持

### Q7：如何切换到公司私有部署大模型？

```bash
$env:LLM_API_KEY="公司私有key"
$env:LLM_BASE_URL="https://公司私有部署地址/v1"
$env:LLM_MODEL="公司模型名"
python -m backend.app
```

只要私有部署兼容 OpenAI 接口协议即可无缝切换。

### Q8：多个业务用户能同时使用吗？

可以。Flask 开发模式支持并发请求，每个会话独立隔离。生产环境建议用 gunicorn/uwsgi 部署。

### Q9：合同审核报"LLM 未配置"？

合同审核模块独立读取环境变量，需单独配置：

```bash
$env:LLM_API_KEY="你的key"
$env:LLM_BASE_URL="https://ark.cn-beijing.volces.com/api/coding/v3"
$env:LLM_MODEL="ark-code-latest"
python -m backend.app
```

注意：合同审核模块不会回退到纯 RAG（与业务咨询不同），必须有 LLM 才能运行审核流水线。

### Q10：合同审核报"business review failed: 'str' object has no attribute 'get'"？

这是 LLM 返回畸形 JSON 导致的，已在最新版本修复。系统会跳过非对象元素并记录警告，审核流程继续执行。如仍频繁出现，建议更换更稳定的 LLM 模型。

### Q11：如何使用法律条文搜索 API？

系统提供三个 API 端点，无需登录即可调用：

**法条搜索**：
```bash
curl -X POST http://127.0.0.1:5000/api/law/search \
  -H "Content-Type: application/json" \
  -d '{"query": "民法典关于合同违约金怎么规定", "num_results": 5}'
```

**法条精确验证**：
```bash
curl -X POST http://127.0.0.1:5000/api/law/verify \
  -H "Content-Type: application/json" \
  -d '{"law_name": "民法典", "article_num": "第143条"}'
```

**法律名称查询**：
```bash
curl -X POST http://127.0.0.1:5000/api/law/lookup \
  -H "Content-Type: application/json" \
  -d '{"keyword": "劳动合同法", "only_valid": true}'
```

注意：法律条文搜索基于国家法律法规数据库 (flk.npc.gov.cn)，需网络连接正常。

### Q12：如何删除桌面上的临时克隆目录？

```bash
Remove-Item -Recurse -Force "D:\Users\Z\Desktop\law-search-temp"
```

### Q13：合同太大审核失败？

合同超过 16K tokens 时系统会自动分块审查。如仍报"上下文超长"错误，请检查：
- 合同是否包含大量表格/图片（建议先转为纯文本）
- LLM 模型的上下文窗口是否过小（建议 ≥ 16K）

### Q14：file_review 模块可以单独运行吗？

可以。该模块完全独立，不依赖 `ai`/`knowledge`/`business` 等其他模块。只需：

1. 配置 `LLM_API_KEY`/`LLM_BASE_URL`/`LLM_MODEL` 环境变量
2. 在 Flask app 中注册蓝图：`app.register_blueprint(file_review_routes.bp)`
3. 即可访问 `/api/file_review/upload`、`/run`、`/report/<sid>`、`/chat` 等端点

详见 [03-详细设计.md#3.8](./03-详细设计.md) 章节。
